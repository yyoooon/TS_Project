### 구현 내용
- 클래스로 구현한 **베이스 컴포넌트를 기반**으로 공통 메소드와 프로퍼티를 확장하여 컴포넌트 생성
- 리액트 컴포넌트 라이프 사이클과 상태기반 렌더링 개념 도입
- 렌더링 시 상태를 의존하고, innerHTML+템플릿 리터럴을 사용하면서 특정 자식 컴포넌트 부분 렌더링 가능
- HistoryAPI로 라우터 기능 구현 -> 각 미니 프로젝트의 이름 클릭 시 페이지 전환
- Fetch api로 서버 통신
